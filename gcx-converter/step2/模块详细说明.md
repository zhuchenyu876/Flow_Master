# Step2 模块详细说明

本文档详细讲解 step2 文件夹中所有模块的功能、职责和实现逻辑。

---

## 1. converter.py - 主转换器

### 职责
**核心转换器，协调所有子模块完成从 Dialogflow 到 Agent Studio 的完整转换**

### 主要功能

#### 1.1 初始化 (`__init__`)
- 加载意图映射 (`intents_mapping`)
- 加载意图参数映射 (`intent_parameters_map`)
- 初始化节点计数器、变量计数器
- 设置语言配置

#### 1.2 主转换方法 (`convert_to_multiple_workflows`)
**核心流程：**
```
1. 解析 Flow 数据
   ├─ 提取 Flow 槽位 (flow_slots)
   ├─ 提取 Flow 条件 (flow_conditions)
   └─ 获取所有 transitionEvents

2. 为每个 triggerIntent 生成独立 Workflow
   ├─ 提取该 Intent 对应的 Flow 条件
   ├─ 生成 start 节点
   ├─ 构建 Flow 槽位抽取链（如果有）
   ├─ 构建 Flow 条件链（如果有）
   ├─ 添加 Intent 参数抽取链（如果有参数）
   └─ 处理 Page 流程

3. 处理每个 Page
   ├─ 生成 Text 节点（responses）
   ├─ 生成 Code 节点（setParameterActions）
   ├─ 生成 Page 槽位抽取链（如果有 slots）
   └─ 处理 transitionEvents（生成条件节点、Intent识别链等）

4. 后处理
   ├─ 过滤无效边
   ├─ 解析 page_xxx 引用
   └─ 去重边
```

#### 1.3 关键方法

**`_generate_single_intent_workflow`**
- 为单个 Intent 生成完整的 Workflow
- 组装所有节点和边
- 处理 Flow 层级和 Page 层级的逻辑

**`generate_workflow_from_page`**
- 从单个 Page 生成节点和边
- 调用 `page_processor.py` 的各个函数
- 处理 Page 的 responses、setParameterActions、transitionEvents

**`_find_jump_node_for_target`**
- 查找或创建 jump 节点
- 处理跨 Flow 或跨 Page 的跳转

### 关键特性
- ✅ 支持为每个 Intent 生成独立的 Workflow
- ✅ 支持 Flow 层级的槽位和条件处理
- ✅ 支持 Page 层级的各种节点生成
- ✅ 支持 Intent 参数抽取
- ✅ 支持多种 transitionEvent 模式

## 2. page_processor.py - Page 层级处理器

### 职责
**处理 Page 层级的节点生成，包括 Text、Code、Condition 等节点**

### 主要函数

#### 2.1 `parse_responses`
**功能：** 解析 Page 的 responses，生成 textReply 节点

**处理流程：**
```
1. 从 page.onLoad.staticUserResponse.candidates 或 page.onLoad.responses 获取响应
2. 筛选指定语言的 responses
3. 转换变量引用：$session.params.xxx → {{xxx}}
4. 生成 textReply 节点
   - type: "textReply"
   - payload: JSON字符串（包含 text, type, buttons 等）
```

**关键逻辑：**
- 支持两种数据结构（原始格式和 step1 处理后的格式）
- 使用 `convert_payload_variables` 转换变量引用
- 保留完整的 payload 内容（包括 buttons、urls 等）

#### 2.2 `parse_parameter_actions`
**功能：** 解析 Page 的 setParameterActions，生成 code 节点

**处理流程：**
```
1. 从 page.onLoad.setParameterActions 获取参数动作
2. 对每个 action：
   - parameter: 参数名
   - value: 参数值（可能包含表达式）
3. 使用 parse_dialogflow_value 解析 value
4. 生成 Python 代码：parameter = value_code
5. 生成 code 节点
   - type: "code"
   - code: 生成的 Python 代码
   - outputs: 输出变量列表
   - args: 输入变量列表
```

**示例：**
```python
# Dialogflow:
setParameterActions: [
  {parameter: "userName", value: "$session.params.name"},
  {parameter: "age", value: "$sys.func.ADD($session.params.birthYear, 2024)"}
]

# 生成的 code 节点：
code = """
userName = name
age = (birthYear + 2024)
"""
outputs = ["userName", "age"]
args = ["name", "birthYear"]
```

#### 2.3 `parse_transition_events`
**功能：** 解析 Page 的 transitionEvents，生成意图识别和条件判断节点

**支持的四种模式：**

**模式1：有 Intent 无 Condition**
```
capture → kb → code(extract intent) → condition → target_pages
```

**模式2：有 Intent 有 Condition（链式判断）**
```
capture → kb → code(extract intent)
  ↓
intent_check1(是否intent1?)
├─ 是 → LLM(提取condition变量) → CODE(解析) → param_condition → target_pages
└─ 否 → intent_check2(是否intent2?) → ...
```

**模式3：纯条件分支（无 Intent）**
```
condition → target_pages
```

**模式4：Intent 单独抽取 + 多个 Condition 分支**
```
capture → kb → code(extract intent) → check_intent
├─ 是 → llm(提取参数) → code(解析) → param_condition → target_pages
└─ 否 → fallback
```

**关键逻辑：**
- 解析条件表达式（支持 AND/OR 多条件）
- 处理 beforeTransition 的 setParameterActions 和 responses
- 生成条件节点和条件分支

#### 2.4 `parse_before_transition_responses`
**功能：** 解析 beforeTransition 中的 staticUserResponse，生成 text 节点

**处理流程：**
- 复用 `parse_responses` 的核心逻辑
- 在 transition 跳转前显示响应

#### 2.5 `convert_payload_variables`
**功能：** 将 payload 中的 Dialogflow 表达式转换为模板格式

**转换规则：**
- `$session.params.xxx` → `{{xxx}}`
- `$sys.func.GET_FIELD($a, $b)` → `{{variable_name}}`
- 递归处理嵌套的 dict 和 list

**示例：**
```python
# 输入：
{
  "text": "Hello $session.params.userName, your age is $sys.func.ADD($session.params.birthYear, 2024)"
}

# 输出：
{
  "text": "Hello {{username}}, your age is {{birthyear}}"
}
```

#### 2.6 `normalize_condition_value`
**功能：** 标准化条件值，保留 'null' 关键字

**处理逻辑：**
- `None` → `""`
- `"null"` → `'null'`（保留字符串）
- 其他值去除首尾空格

#### 2.7 `generate_setparameter_code_node`
**功能：** 为 setParameterActions 生成 code 节点（用于 transition）

**与 `parse_parameter_actions` 的区别：**
- 用于 transition 的 beforeTransition.setParameterActions
- 节点名称包含 intent_name

## 3. parse_expressions.py - 表达式解析器

### 职责
**将 Dialogflow 表达式转换为 Python 代码，支持 40+ 种系统函数**

### 核心函数

#### 3.1 `parse_dialogflow_value`
**功能：** 解析 Dialogflow 的值表达式，转换为 Python 代码

**支持的表达式类型：**

**1. 基础类型**
- `None` → `'None'`
- `bool` → `str(bool)`
- `int/float` → `str(number)`
- `dict/list` → `json.dumps(value)`

**2. 变量引用**
- `$session.params.xxx` → `xxx`（提取变量名）
- `$request.user-utterance` → `user_utterance`
- 自动收集输入变量到 `input_variables` 列表

**3. 系统函数（40+ 种）**

**数学函数：**
- `ADD(a, b, ...)` → `(a + b + ...)`
- `MINUS(a, b)` → `(a - b)`
- `MULTIPLY(a, b, ...)` → `(a * b * ...)`
- `DIVIDE(a, b, scale?)` → `round(a / b, scale)`
- `ROUND(value, digits?)` → `round(value, digits)`

**字符串函数：**
- `CONCATENATE(a, b, ...)` → `str(a) + str(b) + ...`
- `LEN(text)` → `len(str(text))`
- `LOWER(text)` → `str(text).lower()`
- `UPPER(text)` → `str(text).upper()`
- `MID(text, start, length)` → `str(text)[start-1:start-1+length]`
- `SUBSTITUTE(text, old, new)` → `re.sub(r"old", r"new", str(text))`
- `SPLIT(text, delimiter)` → `re.split(r"delimiter", str(text))`
- `JOIN(delimiter, list, final_delimiter?)` → `delimiter.join(map(str, list))`
- `URL_ENCODE(text)` → `urllib.parse.quote_plus(str(text))`

**日期函数：**
- `NOW()` → `datetime.datetime.now().isoformat()`
- `ADD_DATE(date_obj, amount, unit)` → `__add_date(date_obj, amount, unit)`
- `FORMAT_DATE(date_obj, format, lang?)` → `__fmt_date(date_obj, format, lang)`
- `IS_FUTURE_DATE(date_obj)` → `__is_future_date(date_obj)`
- `IS_PAST_DATE(date_obj)` → `__is_past_date(date_obj)`

**列表函数：**
- `APPEND(list, item1, item2, ...)` → `(list + [item1, item2, ...])`
- `REMOVE(list, item1, item2, ...)` → `[x for x in list if x not in [item1, item2, ...]]`
- `COUNT(list)` → `len(list)`
- `UNIQUE(list)` → `sorted(set(list), key=lambda k: list.index(k))`
- `CONTAIN(item, list)` → `(item in list)`
- `MATCH(list, item)` → `(list.index(item) if item in list else -1)`
- `GET(list, index)` → `(list[index] if 0 <= index < len(list) else None)`

**对象函数：**
- `GET_FIELD(obj, key)` → `(obj.get(key, None))`
- `IDENTITY(value)` → `value`
- `FILTER(data, jsonpath_expr)` → `__filter(data, jsonpath_expr)`
- `NESTED_FIELD(obj, key1, key2, ...)` → `__nested_field(obj, "key1", "key2", ...)`

**类型转换函数：**
- `TO_TEXT(value)` → `str(value)`
- `TO_NUMBER(value)` → `float(value)`
- `TO_OBJECT(json_str)` → `json.loads(json_str)`
- `TO_PHONE_NUMBER(number, region?)` → `__phone(number, region)`

**校验函数：**
- `IS_DATE(date_str, format, lang?)` → `__is_date(date_str, format, lang)`
- `IS_PHONE_NUMBER(number, region?)` → `__is_phone_number(number, region)`
- `IS_CREDIT_CARD_NUMBER(card_number)` → `__is_credit_card_number(card_number)`

**生成函数：**
- `RAND()` → `random.random()`
- `UUID()` → `str(uuid.uuid4())`

**逻辑函数：**
- `IF(condition, true_value, false_value)` → `(true_value if condition else false_value)`
  - 特殊处理：支持条件表达式字符串（如 `"$session.params.xxx=null"`）
  - 自动解析条件中的变量引用

**4. 嵌套函数**
- 递归处理嵌套的函数调用
- 例如：`ADD(MULTIPLY(2, 3), MINUS(10, 4))` → `((2 * 3) + (10 - 4))`

#### 3.2 `_split_function_args`
**功能：** 按逗号分割函数参数，正确处理嵌套括号

**处理逻辑：**
- 跟踪括号深度
- 只在顶层逗号处分割
- 支持嵌套函数调用

**示例：**
```python
# 输入：`"$a, $b, GET_FIELD($obj, $key)"`
# 输出：`["$a", "$b", "GET_FIELD($obj, $key)"]`
```

#### 3.3 工具函数
- `__add_date`: 日期加减运算
- `__fmt_date`: 日期格式化
- `__is_future_date`: 判断是否为未来日期
- `__is_past_date`: 判断是否为过去日期
- `__phone`: 电话号码解析
- `__is_phone_number`: 电话号码验证
- `__is_date`: 日期验证
- `__is_credit_card_number`: 信用卡号验证（Luhn算法）
- `__filter`: JsonPath 过滤
- `__nested_field`: 嵌套字段访问

## 4. intent_chain_builder.py - Intent 链构建器

### 职责
**构建 Flow 层级的 Intent 链，包括槽位抽取链和条件链**

### 主要函数

#### 4.1 `build_flow_slot_chain`
**功能：** 构建 Flow 层槽位抽取链：start → capture → llm → parse

**处理流程：**
```
1. 收集 Flow 槽位信息（displayName, entityType, classType）
2. 生成 capture 节点
   - type: "captureUserReply"
   - variable_assign: "last_user_response"
3. 生成 LLM 节点
   - type: "llmVariableAssignment"
   - prompt_template: 包含槽位列表和实体候选值提示
   - 使用实体候选值增强提示（如果 classType == 'ENUMERATION'）
4. 生成 code 节点（parse）
   - type: "code"
   - code: 解析 LLM 输出的 JSON
   - outputs: 槽位变量列表
```

**关键特性：**
- 支持实体候选值提示（从 `entity_candidates` 获取）
- 多语言支持（根据 `lang` 参数选择候选值）
- 自动生成 JSON 解析代码

**示例：**
```python
# Flow 槽位：["userName", "age"]
# 生成的链：
# capture_flow_slots → llm_flow_slots → parse_flow_slots
# 输出变量：["username", "age"]
```

#### 4.2 `build_flow_condition_chain`
**功能：** 构建 Flow 层条件链：chain_source → condition_node

**处理流程：**
```
1. 过滤条件：
   - 只处理指向当前 workflow target 的条件（如果 filter_by_target=True）
   - 过滤无效条件（无变量、无比较值、true值等）
   - 去重（基于 lhs_var, rhs_normalized, comparator）

2. 生成 condition 节点
   - type: "condition"
   - if_else_conditions: 条件分支列表

3. 为每个分支创建条件边
   - connection_type: "condition"
   - condition_id: 分支ID
   - target_node: page_xxx 或 jump_to_flow_xxx（占位符）

4. 添加 other 分支（fallback）
```

**关键逻辑：**
- 支持 AND/OR 多条件组合
- 规范化条件值（列表排序、去重）
- 条件分支指向 page_xxx 占位符，后续解析为实际节点

**示例：**
```python
# Flow 条件：
# - customerSegment = "premium" → page_premium
# - customerSegment = "standard" → page_standard

# 生成的 condition 节点：
# {
#   "type": "condition",
#   "if_else_conditions": [
#     {"condition_id": "flow_condition_1", "conditions": [{"condition_variable": "customersegment", "comparison_operator": "=", "condition_value": "premium"}]},
#     {"condition_id": "flow_condition_2", "conditions": [{"condition_variable": "customersegment", "comparison_operator": "=", "condition_value": "standard"}]},
#     {"condition_id": "flow_condition_other", "logical_operator": "other"}
#   ]
# }
```

## 5. page_slot_extractor.py - Page 槽位抽取器

### 职责
**处理 Page 层级的槽位抽取，构建 capture → llm → parse 链**

### 主要函数

#### 5.1 `build_page_slot_chain`
**功能：** 根据 Page 的 slots 生成抽槽链（capture → llm → code）

**处理流程：**
```
1. 收集 Page 槽位信息
   - displayName: 槽位名称
   - entityType: 实体类型（className）
   - classType: 类型（ENUMERATION, BUILT_IN_CLASS 等）

2. 生成 capture 节点
   - type: "captureUserReply"
   - variable_assign: "last_user_response"
   - variables: 槽位名称列表

3. 生成 LLM 节点
   - 构建 prompt_template
   - 添加实体候选值提示（如果 classType == 'ENUMERATION'）
   - 从 entity_candidates 获取候选值

4. 生成 code 节点（parse）
   - 解析 LLM 输出的 JSON
   - 提取槽位变量
```

**与 Flow 槽位抽取的区别：**
- Page 槽位抽取在 Page 内部处理
- Flow 槽位抽取在 Workflow 开头处理
- 两者逻辑相同，但节点命名和位置不同

**关键特性：**
- 支持实体候选值提示
- 自动生成 JSON 解析代码
- 处理多语言实体候选值

**示例：**
```python
# Page slots: ["productName", "quantity"]
# 生成的链：
# capture_slot_page_xxx → llm_extract_slot_page_xxx → parse_slot_page_xxx
# 输出变量：["productname", "quantity"]
```

## 6. post_processor.py - 后处理器

### 职责
**后处理逻辑，确保生成的 Workflow 结构正确**

### 主要函数

#### 6.1 `filter_invalid_edges`
**功能：** 过滤掉所有无效的边

**过滤规则：**
- ❌ 自环（source == target）
- ❌ jump 节点作为 source 的边（jump 节点不能有出边）
- ❌ jump 到 jump 的边

**处理逻辑：**
```python
for edge in all_edges:
    if source == target:  # 自环
        continue
    if source in jump_node_names:  # jump 节点作为 source
        continue
    # 保留有效边
```

#### 6.2 `can_add_edge`
**功能：** 检查是否可以添加边

**检查规则：**
- ❌ source == target（自环）
- ❌ source 是 jump 节点
- ❌ source 和 target 都是 jump 节点

#### 6.3 `safe_append_edge`
**功能：** 安全地添加边，在添加之前检查是否允许

**处理流程：**
```
1. 调用 can_add_edge 检查
2. 如果允许，创建边对象并添加到列表
3. 如果不允许，记录警告（如果 verbose=True）
```

#### 6.4 `resolve_page_references`
**功能：** 解析 page_xxx 引用为实际的节点名称

**处理逻辑：**
```python
# 输入边：{"source_node": "node1", "target_node": "page_abc123"}
# page_id_to_entry: {"abc123": "text_node_page_abc123"}
# 输出边：{"source_node": "node1", "target_node": "text_node_page_abc123"}
```

**关键特性：**
- 跳过 jump 节点作为 source 的边
- 如果 page_xxx 未找到，保留原样并记录警告

#### 6.5 `deduplicate_edges`
**功能：** 去重边

**去重规则：**
- 对于非 condition 节点，只保留一条 default 出边
- 对于非 condition 节点，不能有多个 condition 边指向同一个 target
- condition 节点可以有多个出边

**处理逻辑：**
```python
for edge in edges:
    if source not in condition_node_names:
        if connection_type == 'default':
            # 只保留第一条 default 出边
            if source not in source_default_edges:
                source_default_edges[source] = edge
            else:
                # 跳过重复的
                continue
        else:
            # condition 边，检查是否重复 target
            if target in source_condition_targets[source]:
                continue
            source_condition_targets[source].add(target)
    else:
        # condition 节点，允许多个出边
        fixed_edges.append(edge)
```

## 7. flow_utils.py - Flow 工具函数

### 职责
**提取 Flow 层级的信息（槽位、条件）**

### 主要函数

#### 7.1 `extract_flow_slots`
**功能：** 收集 Flow 层级的槽位信息（用于 start page 抽槽）

**处理流程：**
```
1. 从 flow.flow.slots 获取槽位列表
2. 提取每个槽位的信息：
   - displayName: 槽位名称
   - entityType: 实体类型（className）
   - classType: 类型（ENUMERATION, BUILT_IN_CLASS 等）
3. 返回槽位信息列表
```

**返回格式：**
```python
[
    {
        'displayName': 'userName',
        'entityType': '@sys.person',
        'classType': 'BUILT_IN_CLASS'
    },
    ...
]
```

#### 7.2 `extract_flow_conditions`
**功能：** 收集 Flow 层级的条件（start page 条件），带过滤与去重

**处理流程：**
```
1. 从 flow.flow.transitionEvents 获取所有事件
2. 对每个事件：
   - 提取 condition（restriction）
   - 提取 target_page_id 和 target_flow_id
   - 解析 lhs（变量）和 rhs（值）
   - 规范化 rhs（列表排序、去重）
3. 过滤无效条件：
   - 无 lhs 变量
   - 无 comparator
   - rhs 为 true/None
   - lhs 以 "JumpTo_" 开头
4. 去重（基于 lhs_var, rhs_normalized, comparator, target）
5. 返回条件列表
```

**返回格式：**
```python
[
    {
        'comparator': 'EQUALS',
        'rhs': 'premium',
        'lhs_var': 'customersegment',
        'target_page_id': 'page_xxx',
        'target_flow_id': None
    },
    ...
]
```

**关键逻辑：**
- 支持列表类型的 rhs（OR 条件）
- 规范化 rhs 用于去重（列表排序转为元组）
- 基于 target 区分相同条件的不同分支

#### 7.3 `extract_flow_conditions_for_event`
**功能：** 仅针对单个 transitionEvent 提取条件（用于按 intent 生成 workflow）

**与 `extract_flow_conditions` 的区别：**
- 只处理单个 event，不遍历所有 events
- 用于为特定 Intent 提取对应的 Flow 条件
- 逻辑相同，但输入范围更小

## 8. flow_start_handler.py - Flow 起始处理器

### 职责
**解析 Flow 层级逻辑，返回结构化数据（不直接生成 nodes/edges）**

### 主要函数

#### 8.1 `parse_flow_start_data`
**功能：** 解析 Flow 层级的起始数据（slots、conditions、transitionEvents）

**处理流程：**
```
1. 获取 flow.flow 对象
2. 提取 transitionEvents
3. 调用 extract_flow_slots 提取槽位
4. 返回结构化数据
```

**返回格式：**
```python
{
    'flow_slot_infos': List[Dict],  # Flow 槽位信息
    'transition_events': List[Dict],  # Flow 层级的 transitionEvents
    'flow_obj': Dict  # flow.flow 对象
}
```

#### 8.2 `get_flow_conditions_for_event`
**功能：** 获取单个 transitionEvent 对应的 Flow 层级条件

**用途：**
- 为特定 Intent 提取对应的 Flow 条件
- 用于生成 Intent 的 Workflow 时，只包含相关的条件

#### 8.3 `get_all_flow_conditions`
**功能：** 获取所有 Flow 层级的条件（用于全局分析）

**用途：**
- 全局分析 Flow 条件
- 调试和验证

## 9. intent_workflow_builder.py - Intent Workflow 构建器

### 职责
**组装单个 Intent 的完整 Workflow（当前为占位符，逻辑在 converter.py 中）**

### 当前状态

## 关键设计模式

### 1. 函数式设计
- 各模块提供纯函数，输入输出明确
- 易于测试和复用

### 2. 分层处理
- Flow 层级：处理 Flow 的 slots 和 conditions
- Page 层级：处理 Page 的节点生成
- 表达式层级：处理表达式转换

### 3. 占位符模式
- 使用 `page_xxx` 占位符，后续解析为实际节点名
- 使用 `jump_to_flow_xxx` 占位符，后续解析为 jump 节点

### 4. 去重和过滤
- 条件去重（基于变量、值、比较符）
- 边去重（非 condition 节点只保留一条 default 出边）
- 无效边过滤（自环、jump 到 jump 等）

## 注意事项

1. **节点命名**：使用 `gen_unique_node_name` 确保节点名称唯一
2. **变量命名**：将 Dialogflow 变量名转换为 Python 风格（`-` → `_`，小写化）
3. **边连接**：jump 节点不能有出边，只能作为目标节点
4. **条件分支**：支持 AND/OR 多条件组合
5. **表达式解析**：递归处理嵌套函数调用
6. **引用解析**：`page_xxx` 占位符需要在后处理阶段解析为实际节点名
7. **实体候选值**：从 `entity_candidates` 获取，用于增强 LLM 提示
8. **多语言支持**：根据 `lang` 参数选择对应语言的实体候选值
