GoogleCX Workflow Conversion Tool

Convert DialogFlow CX exported flows to Dyna.ai compatible workflow configurations.

🚀 Quick Start

# 1. Prepare input files
# Place exported_flow_*.json files in project root or input/ folder
# Place planning JSON template file in input/ folder

# 2. Configure Step 3 API credentials (Optional, only for KB creation)
# Edit step3_config.py and fill in API credentials

# 3. Run the complete pipeline
python run_all_steps.py

File Input Output Purpose

exported_flow_*.json		 output/step2_workflow_config/
- nodes_config_{intent_name}.json
- edge_config_{intent_name}.json
- generated_workflows.json		 Convert fulfillments to workflows
Generate independent config for each flow-level intent

Features:

  • Multi-Workflow Mode: Create independent workflow for each flow-level triggerIntent
  • Process Flow Level: Parse all intents in flow.flow.transitionEvents
  • Process Page Level: Process configuration of pages in fulfillments
  • Support Four TransitionEvent Scenarios:
    • a. Intent without condition: capture → kb → code(extract intent) → condition → target_pages
    • b. Intent with condition (chained):
      capture → kb → code(extract intent) → intent_check1(is intent1?)
  ├─ Yes → LLM(extract condition variables) → CODE(parse) → param_condition → target_pages
  └─ No → intent_check2(is intent2?) → ...
    • c. Pure conditional branch (no intent): condition → target_pages
    • d. Intent extraction + multiple condition branches (Added 2025-11-30):
      capture → kb → code(extract intent) → check_intent(check if matches intent)
  ├─ Yes → llm(extract params) → code(parse) → param_condition → target_pages
  └─ No → fallback
      Description: First route is intent trigger (no target), subsequent routes are condition checks (check session params like ccla-cccl-01)
  • Supported Node Types:
    • start: Start node
    • captureUserReply: Capture user reply
    • knowledgeAssignment: Knowledge base query
    • code: Code execution node
    • condition: Condition check node
    • textReply: Text reply node
    • llmVariableAssignment: LLM parameter extraction node
    • jump: Jump node
  • Workflow Structure: start → page-level nodes (excluding flow-level intent recognition nodes)

Helper Modules:

  • flow_start_processor.py: Process flow-level intent routing

Matching Logic:

  1. Index matching: If flow_name is intent_10 format, extract index 10, match 10th transitionEvent in exported_flow
  2. Name matching: If flow_name is actual intent name, match by normalized name

Fine-tuning Tool ⭐

Location: 微调工具/step8_config_tuner.py

Function: One-click modify KB thresholds and LLM node configs in JSON files under step8_final directory

Main Features:

  1. KB Threshold Config: Batch modify threshold parameters for all KBs

    • qa_min_threshold: QA minimum threshold
    • correlation_threshold: Correlation threshold
    • reference_questions_threshold: Reference questions threshold
    • diff_threshold: Difference threshold
    • ratio_threshold: Ratio threshold
    • file_comparison_threshold: File comparison threshold
    • qa_comparison_threshold: QA comparison threshold
    • text_comparison_threshold: Text comparison threshold

  2. LLM Node Config: Batch modify model selection for all LLM nodes in workflows

    • basic_chat_model: chat_model in basic config
    • intent_final_llm_model: Final LLM model for intent recognition
    • workflow_llm_nodes: Model for all LLM nodes in workflow
    • intention_info_llm_model: llm_model in intention_info

Usage:

# Command line mode: Modify KB thresholds
python 微调工具/step8_config_tuner.py --thresholds "qa_min_threshold=0.9,correlation_threshold=70"

# Command line mode: Modify LLM config
python 微调工具/step8_config_tuner.py --models "workflow_llm_nodes=azure-gpt-4o"

# Modify both thresholds and models
python 微调工具/step8_config_tuner.py --thresholds "correlation_threshold=70" --models "workflow_llm_nodes=azure-gpt-4o"

# Interactive mode
python 微调工具/step8_config_tuner.py --interactive

# Specify input directory
python 微调工具/step8_config_tuner.py --input_dir output/step8_final --thresholds "correlation_threshold=70"

Notes:

  • Tool modifies original files directly, backup step8_final directory before use
  • Supports batch processing of all .json files in specified directory
  • Tool displays modification details for each file and final statistics

Layout Tool ⭐

Location: 布局工具/step7_layout_optimizer.py

Function: Use graph layout algorithm (based on dagre) to beautify node positions in generated workflow JSON files, making workflow diagrams clearer and more readable

Main Features:

  • ✅ Support two layout directions:
    • LR (Left to Right): Layout from left to right (default)
    • TB (Top to Bottom): Layout from top to bottom
  • ✅ Auto-calculate node hierarchy: Use BFS algorithm to calculate node levels
  • ✅ Optimize node spacing: Avoid overlap, auto-adjust node positions
  • ✅ Auto-set connection points: Automatically set node source/target positions
  • ✅ Support batch processing: Support processing single file or entire directory

Usage:

# Process single file (left to right layout)
python 布局工具/step7_layout_optimizer.py --file "output/step6_final/generated_workflow_*.json" --direction LR

# Process single file (top to bottom layout)
python 布局工具/step7_layout_optimizer.py --file "output/step6_final/generated_workflow_*.json" --direction TB

# Batch process all files in directory
python 布局工具/step7_layout_optimizer.py --input "output/step6_final" --direction LR

# Specify output directory
python 布局工具/step7_layout_optimizer.py --input "output/step6_final" --output "output/step7_final" --direction LR

Dependencies:

  • Node.js (for running dagre layout algorithm)
  • @dagrejs/dagre (will auto-install)

Notes:

  • Script overwrites original files by default, backup before processing
  • For complex workflows, recommend using LR (left to right) layout
  • For simple workflows, can use TB (top to bottom) layout
  • Step 7 cleaning isolated nodes automatically calls layout tool for optimization

🔧 Troubleshooting

Issue 1: “Cannot find step3_config.py”

Solution: Ensure step3_config.py exists and API credentials configured (only needed for Step 3)

Issue 2: “Parameter verification failed”

Solution:

  • Check KB description length ≤ 128 characters
  • Confirm embedding parameters are correct

Issue 3: Some workflow files not updated

Solution: Check if kb_per_intent_results_{lang}.json contains that intent

Issue 4: positive_examples empty in Step 8

Solution:

  • Check if input/exported_flow_*.json file exists
  • Check if output/step1_processed/intents_en.json file exists
  • View debug info to confirm matching logic is correct

Issue 5: Step 8 output filename incorrect

Solution:

  • Ensure exported_flow_*.json file is in project root or input/ folder
  • Check if filename format is correct (exported_flow_{project_name}.json)

Issue 6: Step 9 upload failed

Solution:

  • Check if API credentials in .env file are correct (ROBOT_KEY, ROBOT_TOKEN, USERNAME)
  • Check if STEP_9_BASE_URL is correct
  • Check network connection and API service availability
  • View log files in output/step9_upload_logs/ for detailed error info

Issue 7: Layout tool cannot run

Solution:

  • Ensure Node.js is installed (download from https://nodejs.org/)
  • Check if step7_layout_optimizer_dagre.js file exists in 布局工具/ directory
  • If dagre not installed, tool will auto-install, if failed manually run: npm install @dagrejs/dagre

📝 Notes

  1. First run: Recommend setting MAX_INTENTS_TO_PROCESS = 10 for testing (Step 3)

    • Max KB name 255 characters

  4. Template file: Step 8 requires placing planning JSON template file in input/ folder
